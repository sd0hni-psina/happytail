# 📘 Техническое задание: Платформа усыновления животных

---

## 1. Общие требования

### 1.1 Тип системы
Web-платформа с возможностью последующего расширения до мобильного приложения.

### 1.2 Архитектура
- Backend: REST API
- БД: PostgreSQL
- Аутентификация: JWT
- Хранение файлов: S3 или аналог

---

## 2. Роли пользователей

### 2.1 User
- регистрация и авторизация
- просмотр контента
- создание объявлений
- отправка заявок

### 2.2 Shelter Admin
- управление приютом
- управление животными
- обработка заявок
- управление сотрудниками

### 2.3 Shelter Helper
- помощь в управлении животными
- участие в обработке заявок

### 2.4 Platform Admin
- создание приютов
- модерация
- управление системой

---

## 3. Модули системы

---

## 3.1 Аутентификация

### Функции:
- регистрация (email + пароль)

- logout

## 3.2 Пользователи

### Данные:

- имя
- email

### Данные:

- список сотрудников

## 4. Бизнес-логика

---

### 4.1 Создание приюта
- только через администратора платформы

---

### 4.2 Добавление животного
- пользователь или приют
- обязательная валидация данных

---

### 4.3 Усыновление

Flow:
1. создание заявки
2. уведомление приюта
3. изменение статуса
4. коммуникация
5. завершение

---

### 4.4 Волонтёрство

Flow:
1. создание задания
2. выполнение пользователем
3. проверка
4. начисление баллов

---

## 5. API (пример)

### Auth:
- POST /auth/register
- POST /auth/login

### Animals:
- GET /animals
- POST /animals

### Applications:
- POST /applications
- GET /applications

### Chats:
- GET /chats
- POST /messages
